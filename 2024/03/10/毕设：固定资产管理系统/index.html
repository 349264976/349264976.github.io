毕设：固定资产管理系统

开发记录day01：

开发前置

0.0.0：技术选型 -》

​ 前端 ：vue3+vite+element+ruoyiprovue

​ 后端： springboot+mysql+redis+ruoyipromaster

​ 版本控制：git

​ 仓库存储：gitee

1.0.0搭建基础环境ruoyipro

后端搭建：

  1. https://github.com/YunaiV/ruoyi-vue-pro 若以pro的github地址

  2. 使用该地址去 git clone 下载该项目

  3. 使用idea打开项目

  4. 配置maven刷新maven

  5. 查看目录结构修改后端配置

  6. 配置数据库首先初始化项目基础数据

    6.1 初始化数据库

    以mysql为例 初始化 MySQL项目使用 MySQL 存储数据，所以需要启动一个 MySQL 服务，建议使用 5.7 版本。

    ① 创建一个名字为 ruoyi-vue-pro 数据库，【只要】 执行对应数据库类型的 sql (opens new window)目录下的 ruoyi-vue-pro.sql SQL 文件，进行初始化。

    ② 默认配置下，MySQL 需要启动在 3306 端口，并且账号是 root，密码是 123456。如果不一致，需要修改 application-local.yaml 配置文件。

    6.2 初始化redis

    项目使用 Redis 缓存数据，所以需要启动一个 Redis 服务。默认配置下，Redis 启动在 6379 端口，不设置账号密码。如果不一致，需要修改 application-local.yaml 配置文件。

    修改完成后发现代码启动如果出现报错问题

    关闭后端的多租户选项 —-后面开启

  7. 启动项目

    yudao-server (opens new window)是后端项目，提供管理后台、用户 APP 的 RESTful API 接口。

    后端启动项目

至此 后端项目已经启动成功

Daytwo

基本数据库框架构建

含有ruoyipro的基础数据库 还有 毕设的基础数据库

fixed_asset_management_system.sql

  1. 资产表（assets）：
CREATE TABLE assets (
  asset_id INT PRIMARY KEY AUTO_INCREMENT COMMENT '资产ID',
  asset_code VARCHAR(50) NOT NULL COMMENT '资产编码',
  main_asset_code VARCHAR(50) COMMENT '主资产编码',
  asset_name VARCHAR(100) NOT NULL COMMENT '资产名称',
  rfid VARCHAR(50) COMMENT 'RFID标签',
  asset_category VARCHAR(50) NOT NULL COMMENT '资产类别',
  specification VARCHAR(100) COMMENT '规格型号',
  unit VARCHAR(50) COMMENT '计量单位',
  amount DECIMAL(10, 2) COMMENT '数量',
  company VARCHAR(100) COMMENT '所属公司',
  department VARCHAR(100) COMMENT '所属部门',
  user VARCHAR(100) COMMENT '使用人',
  short_region_name VARCHAR(50) COMMENT '所属地区（简称）',
  region_name VARCHAR(100) COMMENT '所属地区',
  location VARCHAR(100) COMMENT '位置',
  manager VARCHAR(100) COMMENT '责任人',
  owner_company VARCHAR(100) COMMENT '归属公司',
  purchase_date DATE COMMENT '购买日期',
  supplier VARCHAR(100) COMMENT '供应商',
  usage_period INT COMMENT '使用期限',
  source VARCHAR(100) COMMENT '来源',
  created_by VARCHAR(100) COMMENT '创建人',
  created_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  modified_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '修改时间',
  status VARCHAR(50) COMMENT '状态'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='资产表';
  1. 入库记录表（receipts）：
CREATE TABLE receipts (
  receipt_id INT PRIMARY KEY AUTO_INCREMENT COMMENT '入库记录ID',
  asset_id INT NOT NULL COMMENT '资产ID',
  receipt_date DATE COMMENT '入库日期',
  receipt_quantity INT COMMENT '入库数量',
  receipt_person VARCHAR(100) COMMENT '入库人',
  FOREIGN KEY (asset_id) REFERENCES assets(asset_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='入库记录表';
  1. 领用记录表（issue_records）：
CREATE TABLE issue_records (
  issue_id INT PRIMARY KEY AUTO_INCREMENT COMMENT '领用记录ID',
  asset_id INT NOT NULL COMMENT '资产ID',
  issue_date DATE COMMENT '领用日期',
  issue_quantity INT COMMENT '领用数量',
  issue_person VARCHAR(100) COMMENT '领用人',
  FOREIGN KEY (asset_id) REFERENCES assets(asset_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='领用记录表';
  1. 退库记录表（return_records）：
CREATE TABLE return_records (
  return_id INT PRIMARY KEY AUTO_INCREMENT COMMENT '退库记录ID',
  asset_id INT NOT NULL COMMENT '资产ID',
  return_date DATE COMMENT '退库日期',
  return_quantity INT COMMENT '退库数量',
  return_person VARCHAR(100) COMMENT '退库人',
  FOREIGN KEY (asset_id) REFERENCES assets(asset_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='退库记录表';
  1. 借用记录表（loan_records）：
CREATE TABLE loan_records (
  loan_id INT PRIMARY KEY AUTO_INCREMENT COMMENT '借用记录ID',
  asset_id INT NOT NULL COMMENT '资产ID',
  loan_date DATE COMMENT '借用日期',
  loan_quantity INT COMMENT '借用数量',
  loan_person VARCHAR(100) COMMENT '借用人',
  return_date DATE COMMENT '归还日期',
  return_quantity INT COMMENT '归还数量',
  return_person VARCHAR(100) COMMENT '归还人',
  FOREIGN KEY (asset_id) REFERENCES assets(asset_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='借用记录表';
