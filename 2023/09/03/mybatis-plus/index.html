这里也提一嘴parameterType=”String”

如果是一个参数我们可以省略不写

如果是多个参数我们可以使用map或者实体类来传递

parameterType=”java.util.Map”

parameterType=”com.po.Student”

我们也可以省略不写 因为mybatis底层帮我们自动映射到

#{studentno} 这里可以是实体的名字 也可以是map的key

但是我们需要注意的是当我们的map中的key和我们#{sno}不一致的时候我们需要在传递过来的时候用@param（”指定名称”）

如果你的实体类或者Map中的键（key）与SQL语句中的占位符（#{value}）的名称不相同，你可以使用@Param注解（对于Java）或者指定键（key）的方式（对于Map）来映射它们。

对于Java实体类，你可以在方法参数上使用@Param注解来指定实体类中属性与占位符的对应关系。例如：

public void deleteStudent(@Param("studentId") String id, @Param("studentName") String name) {
  // 执行删除学生的逻辑
}

然后，在XML映射文件中，你可以使用指定的名称来引用这些参数：

<delete id="deleteStudent">
  DELETE FROM students
  WHERE id = #{studentId}
    AND name = #{studentName}
</delete>

对于Map，你可以在传递参数时，指定键与占位符的对应关系。例如：

Map<String, Object> parameters = new HashMap<>();
parameters.put("idParam", id);
parameters.put("nameParam", name);

deleteStudent(parameters);

然后，在XML映射文件中，你可以使用指定的键来引用这些参数：

<delete id="deleteStudent">
  DELETE FROM students
  WHERE id = #{idParam}
    AND name = #{nameParam}
</delete>

具体的实体我们也可以这样对应

Java方法定义：

public void deleteStudent(Student student) {
  // 执行删除学生的逻辑
}

XML映射文件：

<delete id="deleteStudent">
  DELETE FROM students
  WHERE id = #{student.studentId}
    AND name = #{student.studentName}
</delete>

在这个示例中，你可以创建一个名为Student的Java类，其中包含studentId和studentName等属性，并将其作为参数传递给deleteStudent方法。在XML映射文件中，你可以使用#{student.studentId}和#{student.studentName}来引用Student对象的属性。

注意我们查询的结果mybatis默认不会帮我们做驼峰映射

如果你在 MyBatis 的配置文件中设置了 mapUnderscoreToCamelCase 为 true，并且没有使用自定义的 <resultMap> 配置，那么在没有其他特殊配置的情况下，查询结果的字段将按照驼峰命名规则映射到 Map 中的键名。

假设数据库表中的字段和查询结果如下：

数据库表字段：custom_key, column1, column2

查询结果数据：1, value1, value2

使用以下的配置：

<configuration>
  <settings>
    <setting name="mapUnderscoreToCamelCase" value="true"/>
  </settings>
  <!-- 其他配置 -->
</configuration>

查询结果将会映射到 Map 中的键名为驼峰命名规则，即：

Map<String, Object> resultMap = new HashMap<>();
resultMap.put("customKey", 1);
resultMap.put("column1", "value1");
resultMap.put("column2", "value2");

如果你的实体类中的属性采用了驼峰命名规则，并且希望查询结果按照驼峰命名规则映射到实体类中的属性，那么你需要在 MyBatis 的配置文件中开启驼峰配置。

假设你有一个实体类 Student，其中包含属性 studentNo、studentName 和 studentAge，对应数据库表中的字段 student_no、student_name 和 student_age。

在 MyBatis 的配置文件中添加以下配置：

<configuration>
  <settings>
    <setting name="mapUnderscoreToCamelCase" value="true"/>
  </settings>
  <!-- 其他配置 -->
</configuration>

通过设置 mapUnderscoreToCamelCase 为 true，MyBatis 将会自动将查询结果的字段按照驼峰命名规则映射到实体类的属性。

使用以下的映射配置：

<resultMap id="studentResultMap" type="com.example.Student">
  <id column="student_no" property="studentNo" />
  <result column="student_name" property="studentName" />
  <result column="student_age" property="studentAge" />
  <!-- 其他字段映射规则 -->
</resultMap>

<select id="selectStudent" resultMap="studentResultMap">
  SELECT student_no, student_name, student_age FROM your_table
</select>

在这个示例中，查询结果的字段 student_no 将映射到实体类的属性 studentNo，student_name 映射到 studentName，student_age 映射到 studentAge。

因此，你可以直接将查询结果映射到实体类 Student 的对象中：

Student student = sqlSession.selectOne("selectStudent");

MyBatis 将会自动将查询结果的字段值赋值给实体类的属性，根据驼峰命名规则进行映射。

myabatisplus分页插件

CV 大法好

首先 mybatisplus的分页插件基于 mybatis的 interceptor

先注册好核心拦截器 配置好mp的拦截器配置

    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor(){
        // 1.创建核心拦截器
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        // 2.创建插件
        PaginationInnerInterceptor paginationInnerInterceptor = new PaginationInnerInterceptor(DbType.MYSQL);
        paginationInnerInterceptor.setMaxLimit(1000L);
        interceptor.addInnerInterceptor(paginationInnerInterceptor);
        // 返回
        return interceptor;
    }

注意mybatis的分页插件条件时page对象

返回的数据也是page对象

    @Test
    void testPageQuery() {

        System.out.println("pages = " + pages);
        List<User> records = page.getRecords();

    }
